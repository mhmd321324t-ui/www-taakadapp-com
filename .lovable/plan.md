

# خطة: ترجمة شاملة للتطبيق حسب لغة الجهاز + تبديل التقويم هجري/ميلادي

## نطاق العمل

### المشكلة الحالية
1. **الترجمة**: نظام i18n يترجم فقط مفاتيح UI النصية عبر Google Translate. المحتوى الثابت (أسماء الفئات، الأدعية، القصص، أزرار الوصول السريع) مكتوب بالعربية مباشرة في الكود (hardcoded)
2. **القرآن**: يعرض النص العربي فقط بدون ترجمة للمعنى بلغة المستخدم
3. **التقويم**: يعرض التقويم الهجري فقط بدون خيار للتبديل للميلادي

### الحل المقترح

## الجزء 1: ترجمة المحتوى الثابت في الكود

### `src/pages/Index.tsx`
- تحويل `quickAccessItems` لاستخدام مفاتيح `t()` بدل النص العربي المباشر

### `src/pages/Duas.tsx`
- تحويل أسماء فئات الأدعية (`dailyCategories`, `adhkarCategories`, إلخ) لاستخدام `t()` 

### `src/pages/Stories.tsx`
- تحويل `CATEGORIES` و `MEDIA_TYPES` لاستخدام `t()`

### `src/pages/Ruqyah.tsx`
- الفئات تأتي من قاعدة البيانات بالعربية (`name_ar`) — سنستخدم `name_en` للغات غير العربية (الحقل موجود بالفعل)

### `src/pages/Quran.tsx`
- أسماء السور تأتي من API بالعربية والإنجليزية — سنعرض `englishName` للغات غير العربية

### `src/lib/i18n.ts`
- إضافة ~40 مفتاح ترجمة جديد لكل النصوص الثابتة المكتوبة بالعربية في الصفحات أعلاه

## الجزء 2: ترجمة معاني القرآن حسب لغة المستخدم

### `src/pages/SurahView.tsx`
- عند لغة غير عربية: جلب ترجمة المعنى من AlQuran Cloud API (مثلاً `en.asad` للإنجليزية، `de.aburida` للألمانية)
- عرض النص العربي أولاً ثم الترجمة أسفله
- إضافة mapping للغات → أسماء طبعات الترجمة المتاحة في API

### قائمة الترجمات المتاحة (أمثلة):
- `en` → `en.asad` (Muhammad Asad)
- `fr` → `fr.hamidullah`
- `de` → `de.aburida`
- `tr` → `tr.diyanet`
- `ur` → `ur.jalandhry`
- `id` → `id.indonesian`
- `es` → `es.cortes`

## الجزء 3: تبديل التقويم هجري/ميلادي

### `src/components/HijriCalendar.tsx`
- إضافة state `calendarType: 'hijri' | 'gregorian'`
- إضافة أزرار تبديل (tabs) بين "هجري" و "ميلادي"
- عند اختيار "ميلادي": عرض تقويم ميلادي عادي مع تمييز اليوم الحالي والمناسبات الإسلامية
- الاحتفاظ بالمناسبات الإسلامية في كلا العرضين
