

# إصلاح أوقات الصلاة والمساجد

## المشكلة الحقيقية

بعد فحص الكود والـ API:

1. **أوقات الصلاة العامة تعمل بشكل صحيح** — الـ API يُرجع أوقات صحيحة (الفجر 05:03، الظهر 12:38، العصر 15:38، المغرب 18:22، العشاء 20:08 لأوسنابروك). الفرق بين يوم ويوم هو 1-2 دقيقة فقط.

2. **خطأ في `useSavedMosqueTimes`**: عند حفظ الأوقات يدوياً من صفحة المساجد، تُحفظ بصيغة `{ _date: "...", times: { fajr: "..." } }`. لكن عند القراءة في الصفحة الرئيسية، الكود يبحث عن `times.fajr` مباشرة بدل `times.times.fajr` — فلا يجدها ويتجاهل الأوقات اليدوية.

3. **لا يوجد تحديث تلقائي يومي في الصفحة الرئيسية**: الـ `useEffect` في `useSavedMosqueTimes` يعمل مرة واحدة فقط عند التحميل (`[]` dependency) — لا يُحدّث عند تغيير اليوم.

4. **طريقة الحساب ثابتة**: الأوقات من Aladhan للمساجد تستخدم دائماً `method=3` بدل طريقة حساب المستخدم حسب بلده.

## الحل

### `src/hooks/useSavedMosqueTimes.tsx`
- إصلاح قراءة الأوقات اليدوية لدعم صيغة `{ _date, times }` الجديدة + التحقق من التاريخ
- إضافة تحديث تلقائي عند تغيير اليوم (midnight refresh مثل usePrayerTimes)
- استخدام طريقة حساب المستخدم بدل القيمة الثابتة

### `src/hooks/usePrayerTimes.tsx`
- إضافة `cache-busting` header لتجنب PWA cache قديم
- لا يحتاج تعديل كبير — يعمل بشكل صحيح

### `vite.config.ts`
- تقليل مدة كاش Aladhan API من 12 ساعة إلى 3 ساعات لضمان تحديث أسرع
